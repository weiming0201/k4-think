# 探索 (explore)

> 碱基旋转：α→υ→γ→π (比→问→看→排)  
> 功能表行：区分层（α行）  
> 信息模式：横向环，外部搜索  
> 理论基础：`skills/k4-think/references/k4_meta.md`

**核心任务**：基于拆分的四维度，向外搜索收集可能性空间。

**输入要求**：拆分路径的全部4步输出（看排比问全序列）

**输出传递**：全部4步思考过程 → 验证路径的完整上下文

## 旋转执行：α→υ→γ→π

### Step 1: 比×α (区分区分层)
**功能表定位**：α行×比较列 = "辨别4个信息收集方向"

基于拆分的四维度，确定搜索方向的差异化策略。

**输入解析**：
```
从拆分全输出中提取：
- 四维度定义：[A/B/C/D各是什么]  
- 验证状态：[S₂ __/3, K3 __/4, 𝔑 __/4, 𝔇 ___]
- 主要张力：[拆分识别的核心冲突]
```

**方向辨别**：
```
### 比×α：搜索方向辨别

| 维度 | 搜索重点 | 信息类型 | 预期收获 |
|---|---|---|---|
| A | [这个维度需要什么信息？] | 案例/数据/理论 | [能解决什么问题？] |
| B | [与A的差异化搜索重点] | 案例/数据/理论 | [独特价值是什么？] |
| C | [第三维度的独立方向] | 案例/数据/理论 | [填补什么空白？] |  
| D | [最容易被忽略的信息] | 案例/数据/理论 | [防范什么盲点？] |

**方向正交性**：四个搜索方向不能重叠，each维度的信息需求独立
```

**约束**：每个方向的搜索重点必须基于该维度的定义，不能泛化为"更多信息"。

---

### Step 2: 问×α (质疑区分层)  
**功能表定位**：α行×问列 = "写给自己的搜索prompt"

将搜索方向转化为具体的搜索查询。

```
### 问×α：搜索prompt设计

**维度A搜索**：
Query: [具体搜索词汇/短语]
平台: Brave/Exa/Grok [选择最适合的]
期望: [什么样的结果算成功？]

**维度B搜索**：  
Query: [与A差异化的搜索词]
平台: [可能不同平台更合适] 
期望: [与A不重叠的信息类型]

**维度C搜索**：
Query: [第三个独立查询方向]
平台: [根据信息类型选择]
期望: [补完性信息]

**维度D搜索**：
Query: [反直觉/盲点导向查询]  
平台: [可能需要深度分析]
期望: [challenge既有假设]

**质疑checklist**：
- [ ] 四个query真的不重叠？
- [ ] 是否覆盖了问题的主要争议面？  
- [ ] 是否包含了反直觉角度？
- [ ] 搜索词汇是否具体到可执行？
```

**平台选择原则**：
- Brave：广度、事实性、最新信息
- Exa：深度、学术性、语义搜索  
- Grok：分析性、多角度、正交视角

---

### Step 3: 看×α (观察区分层)
**功能表定位**：α行×看列 = "汇总搜索返回的可能性"

执行搜索，汇总结果，识别可能性空间。

```
### 看×α：搜索执行与汇总

**维度A结果**：
[执行搜索，粘贴关键发现]
核心发现：[1-2句总结]
可能性：[这个维度支持哪些可能的答案/方向？]

**维度B结果**：
[执行搜索，粘贴关键发现]  
核心发现：[与A的差异化发现]
可能性：[新的可能性，不与A重叠]

**维度C结果**：
[执行搜索，粘贴关键发现]
核心发现：[第三方向的独立发现]  
可能性：[补完性的可能方向]

**维度D结果**：
[执行搜索，粘贴关键发现]
核心发现：[反直觉/意外发现]
可能性：[challenge初始假设的方向]

**交叉发现**：
- 跨维度一致性：[哪些发现在多个维度得到支持？]
- 跨维度冲突：[哪些发现相互矛盾？]  
- 意外模式：[搜索中出现但未预期的模式？]
```

**汇总原则**：保持维度独立性，避免强行整合。冲突和矛盾也是有价值的发现。

---

### Step 4: 排×α (组织区分层)
**功能表定位**：α行×排列 = "将搜索结果组织成产出结构"

将搜索发现组织成结构化的可能性列表。

```
### 排×α：可能性结构化

**高置信可能性**（多维度支持）：
1. [可能性1]：支持维度[A,B...]，置信度[高/中/低]
2. [可能性2]：支持维度[C,D...]，置信度[高/中/低]

**中等可能性**（单维度强支持）：
3. [可能性3]：主要来自维度[X]，需要进一步验证  
4. [可能性4]：有趣但证据有限

**低置信可能性**（探索性）：
5. [可能性5]：基于薄弱信号，但值得关注
6. [可能性6]：反直觉但有一定支持

**需要深入的方向**：
- [方向1]：信息不足但关键
- [方向2]：存在矛盾，需要厘清
- [方向3]：可能有盲点，需要特定角度验证

**置信度评估**：
- 信息充分度：[搜索是否覆盖了预期范围？]
- 发现质量：[是否有超出预期的洞察？]  
- 维度平衡：[四个维度的信息是否均衡？]
- 挑战强度：[是否challenge了初始假设？]
```

## 完整输出格式

```markdown
## K4 探索：[问题名]

### Step 1: 比×α 搜索方向辨别
[四维度搜索策略表格]

### Step 2: 问×α 搜索prompt设计
[四个查询设计 + 质疑清单]

### Step 3: 看×α 搜索执行与汇总  
[四维度搜索结果 + 交叉发现]

### Step 4: 排×α 可能性结构化
[分级可能性列表 + 深入方向 + 置信评估]

---
**探索总结**：
- 发现数量：高置信__个，中等__个，低置信__个
- 维度覆盖：[A强/B中/C弱/D充分] 
- 主要意外：[最surprising的发现]
- 下步建议：[需要验证什么？需要收敛什么？]
```

## 传递协议

**下游路径接收**：
1. 完整的4步思考过程（比问看排全序列）
2. 结构化的可能性列表（分级）
3. 搜索覆盖评估和发现质量  
4. 需要深入验证的方向

**用途映射**：
- 验证：用可能性列表作为框架填充对象
- 收敛：用探索发现作为质疑材料  

## 搜索执行指南

### 工具选择

```
web_search: query="..." count=5-10
- 适用：事实性、最新信息、广度搜索
- 维度A/B的主流信息收集

Exa API: type="deep" query="..."  
- 适用：学术性、深度分析、语义搜索
- 维度C/D的深度探索

sessions_spawn + Grok:
- 适用：需要分析性思考的查询
- 跨维度模式识别
```

### 反模式预警

❌ **搜索重叠**：四个维度查询雷同，没有区分化策略  
❌ **浅层汇总**：只复制搜索结果，不提炼可能性  
❌ **强行整合**：试图消除维度间的冲突，失去多样性  
❌ **不执行搜索**：基于已有知识推测，破坏外部输入模式  
❌ **忽略反直觉**：只搜索支持既有观点的信息
