# 收敛 (converge)

> 碱基旋转：υ→α→π→γ (问→比→排→看)
> 功能表行：自指层（υ行）
> 职责：内部一致性。框架自洽吗？结论站得住吗？不搜外部信息。
> 元定理：Banach（迭代收敛）、Noether 第一定理（路径无关）、DPI（压缩只丢不增）、Knaster-Tarski（多不动点）、Gödel 不完备性（矛盾是极限不是 bug）

**核心任务**：检查整个 K4 过程的内部一致性。不加新信息，不搜外部，纯内生判断。

**认知定位**：拆=known knowns，探=known unknowns，验=unknown unknowns，收=making it coherent。

**输入要求**：拆+探+验的全部输出

## 旋转执行：υ→α→π→γ

### Step 1: 问×υ (质疑自指层)
**功能表定位**：υ行×问列 = "问题本身对不对"

```
### 问×υ：元质疑

**1. 问题还是那个问题吗？**
拆的时候定义的问题 P，经过探和验之后，P 变了没有？
- 没变 → 继续
- 变了 → 写出新的 P'。如果 P' 和 P 差距大 → 当前分析可能在解错题

**2. 拆的维度经过探验还站得住吗？**
| 维度 | 探有没有挑战 | 验的吻合度 | 还站得住？ |
|---|---|---|---|
| A | | | |
| B | | | |
| C | | | |
| D | | | |

有维度倒了但还在用 → 内部不一致。

**3. Procedure Invariance（Noether 不变量 — 路径无关性）**
如果拆的时候选了不同的张力对开始，最终结论会一样吗？

不重跑。做思想实验：
- 实际路径：从 [X] 张力出发 → 拆出 ABCD → 结论 [Y]
- 假设路径：如果从 [Z] 张力出发 → 会拆出什么？ → 会走到同一个 [Y] 吗？
- 写出具体的分叉点。不能说"应该差不多"

一样 → 结论稳定（Noether 不变量成立）
不一样 → 两种可能：
  a. 结论是路径产物（gauge artifact），不可靠 → 回拆
  b. 不同路径到不同不动点（Knaster-Tarski：单调映射可有多不动点），两个都"对" → 输出两方案
```

---

### Step 2: 比×υ (区分自指层)
**功能表定位**：υ行×比较列 = "读矩阵形状 + 找内部矛盾"

```
### 比×υ：矩阵规律 + 矛盾检测

**0. 发展度矩阵规律**（Noether 同调群：从数字到结构。从验的矩阵中读）

读矩阵的形状，不只看平均分：
- 行均最低是哪行？→ 哪个分析层（γ/π/α/υ）最薄弱
- 列均最低是哪列？→ 哪个问题维度最不清楚
- 对角线(A×γ, B×π, C×α, D×υ)强不强？→ 维度自身理解 vs 交叉理解
- 有没有三角/梯度模式？→ 发展有方向性，顺着推就是下一步
- 有没有孤立的低分格？→ 单点盲点
- 有没有整片低分区？→ 结构性缺口

矩阵形状：[对角线型 / 梯度型 / 孤立低分 / 整片缺口 / 均匀]
最大瓶颈：不是"哪格最低"——是"哪个模式说明了什么问题"
下一步方向：矩阵形状指向哪里？

**1. 维度间矛盾**
A 和 B 的结论矛盾吗？A 和 C 呢？逐对检查。

| 维度对 | 矛盾？ | 具体矛盾点 |
|---|---|---|
| A×B | | |
| A×C | | |
| A×D | | |
| B×C | | |
| B×D | | |
| C×D | | |

矛盾数：___/6
≥ 2 → 框架内部不自洽，需要回拆修维度定义
注意（Gödel 不完备性）：矛盾 = 1 可能是系统极限不是拆的错误。
判断：这个矛盾是"完备性的代价"还是"维度定义错了"？前者标记共存，后者回拆。

**2. 事实间矛盾**
验里标 ● 的事实之间有没有互相矛盾的？

| 事实1 | 事实2 | 矛盾？ |
|---|---|---|
| | | |

有矛盾 → 至少一条是错的或语境不同。标记哪条更可信。

**3. 层间矛盾**
拆说的、探说的、验说的，三者一致吗？

| | 拆说的 | 探说的 | 验说的 | 一致？ |
|---|---|---|---|---|
| 总方向 | | | | |
| 关键判断 | | | | |
| 优先级 | | | | |

不一致 → 某一步出了偏。标记哪步最可能有问题。
```

### Step 3: 排×υ (组织自指层)
**功能表定位**：υ行×排列 = "压缩结论"

```
### 排×υ：压缩与展开测试（Shannon DPI — 处理只丢不增）

**压缩**：
把整个 K4 过程的结论压缩成一句话：
"[一句话结论]"

**压缩一定丢东西（DPI）。标记丢了什么。**

**展开测试**：
从这一句话出发，能不能无损展开回完整过程？

1. 这句话蕴含了哪些维度？→ 和拆的 ABCD 对得上吗？
2. 这句话的每个关键词有事实支撑吗？→ 和验的 ● 对得上吗？
3. 这句话有没有漏掉拆/探/验里的关键发现？
4. 丢掉的部分是可接受的损失还是致命遗漏？

能展开 → 压缩无损。结论稳固。
展不开 → 压缩时丢了东西。丢的那部分可能被无意识忽略了。
```

---

### Step 4: 看×υ (观察自指层)
**功能表定位**：υ行×看列 = "最终置信度"

```
### 看×υ：最终判定

**内部一致性评分**：

| 检查 | 结果 | 影响 |
|---|---|---|
| 问题还是那个问题 | P/P' | |
| 维度都站得住 | ___/4 | |
| 路径无关性 | 通过/不通过 | |
| 维度间矛盾 | ___/6 | |
| 事实间矛盾 | 有/无 | |
| 层间一致性 | 一致/不一致 | |
| 压缩无损 | 是/否 | |

**最终 𝔇**：
基于以上全部检查的综合判断（0-1）

**𝔇 计算约束**：
- 路径无关性不通过 → 𝔇 上限 0.7
- 维度间矛盾 ≥ 2 → 𝔇 上限 0.5
- 压缩有损 → 𝔇 上限 0.8
- 问题变了(P≠P') → 𝔇 上限 0.6

**kernel 探针**（压缩展开之后做）：
逐条列结论的关键判断，每条标记证据类型：
| 判断 | 正面证据支撑 / 无反面证据推定 |
|---|---|
| [判断1] | [哪种？] |
| ... | |

"无反面证据推定" = 结论依赖"没人说不对"而非"有人说对"。这部分𝔇打折。

**多轮对比**（必填，首轮写"首轮，无上轮可比"）：
与上轮对比：[结论变了/没变]
𝔇: ___ (Δ𝔇 = +/- ___)
Cauchy判定：|Δ𝔇| ≤ 0.05 = 收敛（没有显著变化）
```

## 完整输出格式

```markdown
## K4 收敛：[问题名]

### Step 1: 问×υ 元质疑
[问题漂移 + 维度存活 + 路径无关性]

### Step 2: 比×υ 矛盾检测
[维度间 + 事实间 + 层间矛盾]

### Step 3: 排×υ 压缩展开
[一句话结论 + 展开测试]

### Step 4: 看×υ 最终判定
[一致性评分 + 𝔇 + 多轮对比]

---
**收敛总结**：
- 一句话结论：[___]
- 𝔇：___
- 矩阵形状：[___]
- 最大瓶颈：[___]
- 内部矛盾：___个
- 压缩无损：是/否
- 下一步方向：[矩阵指向哪里]

调度全在 `chain.md`（阳）/ `antichain.md`（阴）。

## 标准化输出

1. 矩阵形状 + 最大瓶颈
2. 维度间矛盾数 + Gödel 判断（极限 vs 错误）
3. PI 结果（Noether：通过/不通过/多不动点 Knaster-Tarski）
4. 一句话压缩结论 + 展开测试（DPI）
5. kernel 探针（正面证据 vs 无反面推定 逐条）
6. 𝔇 + 多轮 Δ𝔇

## 反模式预警

❌ **伪质疑**：走形式但不真的质疑。"经过深思熟虑，结论不变" = 没质疑
❌ **搜外部信息**：收不搜。想搜 = 应该回探
❌ **行动方案混入**：收的产出是𝔇和一句话结论，不是行动计划。行动计划在K4之后
❌ **跳过压缩测试**：直接给𝔇不压缩。压缩是收最核心的检验——展不开就不算收完
❌ **无限循环**：𝔇不够就一直转。5轮强制停止
