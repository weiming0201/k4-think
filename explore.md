# 探索 (explore)

> 碱基旋转：α→υ→γ→π (比→问→看→排)
> 功能表行：区分层（α行）
> 信息模式：横向环，外部搜索
> 元定理：Shannon 噪声信道编码（信噪比够则信息可无错提取）、Shannon 信道容量（搜索有上限）、正交基（搜索结果线性无关）、Bayesian update/surprise（信息增量度量）

**核心任务**：基于拆分的四维度，向外搜索收集可能性空间。

**存在性保证（Shannon 噪声信道编码）**：探是有噪声的信道。搜索回来有信号有噪声。但只要搜索的信道容量够（源够多、够正交），信号可以从噪声中提取。三件套（覆盖率/正交性/冲击度）就是在测信道够不够。

**输入要求**：拆分路径的全部4步输出（看排比问全序列）

**输出传递**：全部4步思考过程 → 验证路径的完整上下文

## 旋转执行：α→υ→γ→π

### Step 1: 比×α (区分区分层)
**功能表定位**：α行×比较列 = "辨别4个信息收集方向"

基于拆分的四维度，确定搜索方向的差异化策略。

**输入解析**：
```
从拆分全输出中提取：
- 四维度定义：[A/B/C/D各是什么]  
- 验证状态：[S₂ __/3, K3 __/4, 𝔑 __/4, 𝔇 ___]
- 主要张力：[拆分识别的核心冲突]
```

**四信息源**（固定正交，不随问题P变化）：

```
### 比×α：搜索方向辨别

| 信息源 | K4映射 | 看什么 | 质量标准 | 典型平台 |
|---|---|---|---|---|
| 学术界 | α区分 | 概念边界、taxonomy、已有框架 | 同行审议、有引用网络、优先 primary research 而非综述 | Google Scholar（综述入口→顺引用找原始论文）、arXiv（前沿预印本）、Semantic Scholar（引用图谱追溯 primary） |
| 工业界 | π结构 | **硬实现**：代码、架构、benchmark、可复现的工程产出 | 真跑过、有代码、有数据、**可复现** | GitHub（源码+issue+discussion）、Stack Overflow（工程解的正误投票） |
| 亲历者 | γ存在 | **软经验**：做过这件事的人怎么说。踩坑记录、决策理由、事后复盘 | **本人做过**、有具体细节、非转述 | 技术博客（个人博客 > 企业博客）、conference talks、postmortem。社媒（Reddit/HN）仅作补充 |
| 平行领域 | υ自指 | 同构问题的解法、跨学科类比 | 不同学科、结构同态、问题形状相同但领域不同 | Wikipedia（跨学科术语映射入口）、Google Scholar（换领域术语重搜）、请求外部模型从其他学科视角分析 |

**方向辨别**：基于拆分的四维度，确定每个信息源具体搜什么。

| 信息源 | 针对P的搜索重点 | 预期收获 |
|---|---|---|
| 学术界 | [P在学术上怎么被定义/分类？] | [框架/taxonomy] |
| 工业界 | [P在工程实践中怎么解决？] | [方案/代码/架构] |
| 亲历者 | [做过这件事的人怎么说？踩过什么坑？] | [经验/教训/决策理由] |
| 平行领域 | [什么别的领域遇到过P的同构问题？] | [类比/迁移方案] |
```

**约束**：
- 四个信息源是固定的，不随问题变。变的是每个源里具体搜什么。
- 平行领域（υ）是找正面搜搜不到的东西。换领域的关键词，结构相同但名字不同的问题。

---

### Step 2: 问×α (质疑区分层)  
**功能表定位**：α行×问列 = "写给自己的搜索prompt"

将搜索方向转化为具体的搜索查询。

```
### 问×α：搜索prompt设计

**学术界（α区分）**：
Query: "[P的学术术语] survey OR taxonomy OR classification"
期望: 已有分类框架、定义边界、综述文章
精修: 用领域专用术语，不用日常用语

**工业界（π结构）**：
Query: "[P的工程关键词] implementation OR architecture OR best practice"
期望: 具体方案、踩坑记录、架构图、性能数据
精修: 搜代码时加语言/框架限定词

**亲历者（γ存在）**：
Query: "[P] postmortem OR lessons learned OR 'I built' OR 'we migrated' OR '踩坑'"
期望: 做过这件事的人的第一手经验、决策理由、失败教训
精修: 过滤旁观评论——看帖子里有没有具体细节（时间、数字、代码片段）。有 = 亲历。没有 = 转述，跳过

**平行领域（υ自指）**：
Query: 把P的核心结构用另一个领域的术语重写
期望: 同构问题的解法、类比、对偶结构
精修方法:
  1. 提取P的抽象结构（不是内容，是形状）
  2. 问"什么别的领域有同样形状的问题？"
  3. 用那个领域的术语搜索
  例: "多session记忆同步" → 抽象为 "分布式状态一致性" → 搜分布式系统领域

**质疑checklist**：
- [ ] 四个query用的术语来自不同圈子？（学术术语 ≠ 工程关键词 ≠ 日常说法 ≠ 其他学科的术语）
- [ ] 平行领域的query真的换了领域而不是换了同义词？
- [ ] 亲历者搜索包含了失败案例？
- [ ] 搜索词具体到可直接执行？
```

---

### Step 3: 看×α (观察区分层)
**功能表定位**：α行×看列 = "汇总搜索返回的可能性"

执行搜索，汇总结果，识别可能性空间。

```
### 看×α：搜索执行与汇总

每个信息源记录：

| 信息源 | 有效结果？ | 核心发现 | 信息层级 | 可能性 |
|---|---|---|---|---|
| 学术界 | ✅/❌ | [1-2句] | primary/secondary | [支持什么方向？] |
| 工业界 | ✅/❌ | [1-2句] | primary/secondary | [支持什么方向？] |
| 亲历者 | ✅/❌ | [1-2句] | 亲历/转述 | [支持什么方向？] |
| 平行领域 | ✅/❌ | [1-2句] | — | [什么同构问题？] |

**信息层级标注**（Giljarevskij准则）：
- primary = 原始研究/原始数据/亲历者经验
- secondary = 综述/分析/评论/转述
- 优先采信 primary。全是 secondary = 搜索深度不够

**搜空处理**：
某源返回空 ≠ 失败。先诊断：
- query 太窄？→ 放宽关键词重搜
- 这个领域真的没人讨论？→ 本身是发现（沉默 = 信号）
- 搜的平台不对？→ 换平台
诊断后仍空 → 标记"该源无覆盖"，继续，不阻塞

**交叉发现**：
- 跨源一致：[哪些发现在多个源得到支持？]
- 跨源冲突：[哪些发现相互矛盾？]
- 意外模式：[搜索中出现但未预期的？]
```

**汇总原则**：保持源独立性，避免强行整合。冲突和矛盾也是有价值的发现。

**信道容量警告（Shannon）**：每个源的搜索结果有信息上限。同一个源搜10次不如4个源各搜2次。搜索边际收益递减时停——超过信道容量的部分全是噪声。

---

### Step 4: 排×α (组织区分层)
**功能表定位**：α行×排列 = "将搜索结果组织成产出结构"

将搜索发现组织成结构化的可能性列表。

```
### 排×α：可能性结构化

**高置信可能性**（多维度支持）：
1. [可能性1]：支持维度[A,B...]，置信度[高/中/低]
2. [可能性2]：支持维度[C,D...]，置信度[高/中/低]

**中等可能性**（单维度强支持）：
3. [可能性3]：主要来自维度[X]，需要进一步验证  
4. [可能性4]：有趣但证据有限

**低置信可能性**（探索性）：
5. [可能性5]：基于薄弱信号，但值得关注
6. [可能性6]：反直觉但有一定支持

**需要深入的方向**：
- [方向1]：信息不足但关键
- [方向2]：存在矛盾，需要厘清
- [方向3]：可能有盲点，需要特定角度验证

**置信度评估**：
- 信息充分度：[搜索是否覆盖了预期范围？]
- 发现质量：[是否有超出预期的洞察？]  
- 维度平衡：[四个维度的信息是否均衡？]
- 挑战强度：[是否challenge了初始假设？]
```

|---|---|---|

| 工业×亲历 | | |

正交度：独立信息数 ___/6

#### 框架冲击度（Bayesian update — 新信息改变后验多少）
```

1. [发现X 挑战了 维度A/B/C/D 的定义]
2. ...

冲击数：___
0 → 可能 confirmation bias。换反面关键词重搜

≥3 → 框架可能有结构性问题。考虑回拆
```

1. 四源搜索结果（学术/工业/亲历者/平行领域各自发现）
2. 三件套检验（覆盖率/正交性/冲击度）

## 搜索执行指南

### 执行说明

工具选择参照 `TOOLS.md`。方法论不绑定具体工具。

**需要外部模型视角时**（平行领域υ，或需要正交思考）：
请求外部模型从其他学科视角分析——获取独立于当前底座的正交视角。

### 反模式预警

❌ **搜索重叠**：四个维度查询雷同，没有区分化策略  
❌ **浅层汇总**：只复制搜索结果，不提炼可能性  
❌ **强行整合**：试图消除维度间的冲突，失去多样性  
❌ **不执行搜索**：基于已有知识推测，破坏外部输入模式  
❌ **忽略反直觉**：只搜索支持既有观点的信息
