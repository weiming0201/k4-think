# 拆分 (decompose)

> 碱基旋转：γ→π→α→υ (看→排→比→问)
> 功能表行：存在层（γ行）
> 元定理：Noether 正规化（有限维度够了）、Yoneda（维度由关系定义）、Noether 第二定理（gauge 约束→维度间锁定）

**核心任务**：把问题P拆成四个正交维度并验证拆分合理性。

**存在性保证（Noether 正规化）**：任何有限复杂度的问题都可以用有限个基础维度展开。4个维度不是约定——是正规化。


**输出传递**：全部4步思考过程 → 后续路径作为完整上下文

## 旋转执行：γ→π→α→υ

### Step 1: 看×γ (观察存在层)
**功能表定位**：γ行×看列 = "回看已有的事实和背景"

读取问题P的已知信息，不推测，不搜索外部。

**硬约束**：输出只允许"是什么"、"有什么"、"矛盾在哪"。不允许"应该"、"可能是因为"、"建议"。看到推测或分析 → 砍掉，回到事实。

**输出格式**：
```
### 看×γ：已有观察
**问题核心**：[P的本质是什么？]
**已知事实**：[列举确定的信息]  
**主要张力**：[冲突/矛盾在哪里？]
**边界条件**：[什么在考虑范围内/外？]
```

**约束**：
- 只看当前上下文，不添加外部知识
- 识别张力但不分析，为Step 2做准备
- 不预设维度框架

---

### Step 2: 排×γ (组织存在层)  
**功能表定位**：γ行×排列 = "基于已有事实，用K4拆分问题"

基于Step 1的事实和张力，拆分成4个维度。

**拆分启发**：
1. 从最明显的张力/对立开始 → 第一对{X,Y}
2. X缺什么补完？ → 候选Z  
3. Y缺什么补完？ → 候选W
4. 检验{X,Y,Z,W}是否构成完整空间

**输出格式**：
```
### 排×γ：K4拆分

| 维度 | 定义 | 核心特征 |
|---|---|---|
| A | [具体领域术语] | [这个维度关注什么] |
| B | [具体领域术语] | [与A的根本差异] |  
| C | [具体领域术语] | [隐藏的第三维度] |
| D | [具体领域术语] | [最容易被忽略的维度] |

**拆分逻辑**：[从什么张力开始？为什么是这4个？]
```

**命名约束**：维度必须包含P的领域术语，不能是"准确性/完整性/逻辑性/实用性"等万能词汇。

---

### Step 3: 比×γ (区分存在层)
**功能表定位**：γ行×比较列 = "验证拆分：正交性、S₂配对、K3退化"

三层验证确保拆分质量。

#### 3a. S₂检验（正交性 — Yoneda：维度由关系定义。Shannon 互信息：I(A;B)=0 → 完美正交）

```
### 比×γ：S₂配对验证

| 配对 | 组1特征 | 组2特征 | 检验结果 |
|---|---|---|---|
| {A,B}\|{C,D} | [共同特征] | [共同特征] | ✅/❌ |
| {A,C}\|{B,D} | [共同特征] | [共同特征] | ✅/❌ |  
| {A,D}\|{B,C} | [共同特征] | [共同特征] | ✅/❌ |

**硬标准**：
1. 命名测试：每组特征可用一词命名？
2. 反转测试：标签互换后明显错误？
3. 稳健性：用另一个分组依据，配对结果相同？

通过数：___/3

**Noether 第二定理检查（gauge 约束）**：
三组配对中，有没有一组是"锁定"的——改一边必须改另一边？
锁定的配对 = gauge 约束 = 实际自由度 < 4。
有 → 标记。不影响拆的有效性，但收的时候要知道维度间不完全独立。
```

#### 3a+. 2×2 格子检验（V₄ = Z₂×Z₂ 直积结构）

S₂ 说"能配对"。2×2 更强：配对来自两条独立命名轴。

```
试画 2×2：找两条二元轴 X 和 Y，使得四维度填满四象限。

        Y+        Y-
X+      A         D
X-      B         C

**检验清单**：
- [ ] 两条轴各自有独立的二元语义？（如 内部/外部、扩张/收缩）
- [ ] 对角线互反？A↔C 两轴都反，B↔D 两轴都反
- [ ] 行邻居共享 X 轴？A-D 同行，B-C 同行
- [ ] 列邻居共享 Y 轴？A-B 同列，D-C 同列

画得出 → 四维度是 Z₂×Z₂ 结构（最强正交性）
画不出 → 四维度可能不是真正的四元素。回 Step 2 重新找张力对
```

---

#### 3b. K3退化（必要性 — 维度独立性：去掉一个维度后秩下降）

```
**K3退化测试**：

| 缺失维度 | 三元组 | 退化表现 | 可识别度 |
|---|---|---|---|
| 缺A | BCD | [会变成什么样？] | ✅/❌ |
| 缺B | ACD | [会变成什么样？] | ✅/❌ |
| 缺C | ABD | [会变成什么样？] | ✅/❌ |  
| 缺D | ABC | [会变成什么样？] | ✅/❌ |

通过数：___/4
```

#### 3c. 非显然性𝔑

```
**𝔑检查清单**：

| 检查项 | ✅/❌ | 备注 |
|---|---|---|
| 维度新颖性：在2个已知分类法中找完全相同的维度，找到=不新颖❌，找不到=新颖✅ | | 二值判断，不主观 |
| 第4个元素：首次看到时有"原来如此"感 | | |
| 框架重叠度：与2个已知分类法对比<50%相同项 | | |
| S₂配对：至少1组配对用另一个分组依据仍成立 | | |

checksum：___/4

**判定说明**：
- "新颖性"测试：找2个P领域的已知分类法，逐维度比对。有完全相同的维度=你在重复已有框架
- "原来如此"测试：读后想"我怎么没想到这个角度"→算意外
- "重叠度"测试：找2个相关分类法，数重叠项，<50%→算新颖
- "配对"测试：换一个分组依据（不用原来的标准），看S₂配对结果变不变。不变=配对稳健
```

### Step 4: 问×γ (质疑存在层)
**功能表定位**：γ行×问列 = "自主判断拆分是否合理，输出置信度𝔇"

综合前三步，直觉判断拆分质量。

**𝔇计算**（门控约束）：
```
基础直觉：___ (0-1，完全主观)

门控调整：
- S₂ < 2/3 → 𝔇上限0.5  
- K3 < 3/4 → 𝔇上限0.6
- 𝔑 < 2/4 → 𝔇上限0.7

最终𝔇：___ 

**质量诊断**：
𝔇 ≥ 0.9 → 高确信
𝔇 0.7-0.9 → 可用但有盲点
𝔇 < 0.7 → 不可靠
𝔇 < 0.5 → 框架本身有问题
```

**后验分解**：如果𝔇偏低，哪个验证环节拉低了置信度？

---

### Step 4b: CE反查（穷举性检验 — FMEA 失效模式分析）

S₂验ME（互斥），此步验CE（穷举）。

**操作**：问"这件事怎么死？"列3-5种**单因素致死**模式。每种死法检查是否被某个维度覆盖。

```
### CE反查：死法清单

| # | 死法（单因素致死） | 被哪个维度覆盖？ |
|---|---|---|
| 1 | [具体死法] | [维度X / ❌ 无覆盖] |
| 2 | ... | ... |
| 3 | ... | ... |

未覆盖的死法 = 遗漏的维度 → 补充后回到Step 2重拆
```

**约束**：
- 只列"单因素致死"——需要3个条件同时满足才死的不算
- 3-5种够了，不追求穷举所有可能
- 辅助扫描：用户/监管/供应商/竞争对手/资金方——5个相关方×"他能让你死吗？"
- 发现遗漏 → 补维度 → 重跑S₂/K3 → 更新𝔇

**来源**：2026-03-20 彩票练习——初始拆4维度漏了支付（致命维度），到验才发现。CE反查可在拆阶段提前捕获。

## 反模式预警

❌ **凑四个**：3个够用就别硬凑，K3测试会暴露多余维度  
❌ **万能模板**：用"准确性/完整性"等对任何P都成立的词  
❌ **叙事陷阱**：S₂用感觉而非硬标准（命名/反转/稳健性）  
❌ **跳过验证**：直接输出四维度，不做S₂/K3/𝔑检验  
❌ **越界搜索**：在Step 1添加外部信息，破坏观察纯度
❌ **Step 1 就分析**："可能是因为X导致了Y"出现在 Step 1 → 你在排不在看。回到事实
