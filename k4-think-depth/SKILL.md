# 收敛 (converge)

> 碱基旋转：υ→α→π→γ (问→比→排→看)  
> 功能表行：自指层（υ行）  
> 信息模式：纵向环，内生判断  
> 理论基础：`skills/k4-think/references/k4_meta.md`

**核心任务**：质疑前序分析，调整方向和结构，收敛到可执行行动方案。

**输入要求**：前序路径的全部输出（拆分+探索+验证的完整上下文）

**输出传递**：收敛判断+置信度评估+循环决策

## 旋转执行：υ→α→π→γ

### Step 1: 问×υ (质疑自指层)
**功能表定位**：υ行×问列 = "元质疑：问题本身是否是问题"

质疑整个分析的根本前提和出发点。

**输入汇总**：
```
从前序路径提取核心结论：
- 拆分四维度：[A, B, C, D]，置信度𝔇___
- 探索发现：[高置信__个，中等__个，低置信__个]  
- 验证完备度：[平均___，缺口___个]
```

**元质疑框架**：
```
### 问×υ：元质疑

**问题合理性质疑**：
1. 这个问题真的需要解决吗？[现状vs理想状态的差距是否足够大？]
2. 问题定义是否精确？[是否在解决一个伪问题或表症？] 
3. 解决这个问题的收益是否值得成本？[opportunity cost视角]
4. 问题是否在正确的层次？[是否应该向上/向下一层？]

**分析路径质疑**：
1. K4是否是合适的框架？[这个问题域是否适合四维分析？]
2. 拆分是否抓住了核心？[四维度是否遗漏了关键因素？]
3. 探索是否充分？[搜索是否覆盖了必要信息？] 
4. 验证是否可信？[完备性框架是否有效？]

**时机质疑**：
1. 现在是解决这个问题的合适时机吗？
2. 是否有更紧急的问题需要优先处理？
3. 外部条件是否支持问题解决？

**能力质疑**：
1. 当前资源是否足以解决这个问题？
2. 是否具备必要的知识/技能/权限？
3. 是否有足够的执行时间？

**Procedure Invariance 检查**（诺特检查）：
> 如果这个分析从不同路径/不同维度顺序/不同分析强度开始，核心结论变不变？
> - 变了 → 结论是路径产物（gauge artifact），降低 𝔇，需继续分析
> - 没变 → 结论是规范不变的（gauge invariant），提升 𝔇
> 
> 理论依据：Tversky et al. (1990) procedure invariance；诺特定理 gauge invariance。
> 简化版：换一条路走，到同一个终点吗？

**元质疑结论**：
□ 问题合理且值得继续
□ 问题需要重新定义  
□ 问题不值得解决
□ 时机/能力不合适，暂缓
```

**质疑强度**：如果多个质疑指向根本问题，考虑终止分析或重新开始。

---

### Step 2: 比×υ (区分自指层)
**功能表定位**：υ行×比较列 = "质疑并调整现有方向"

基于元质疑结果，调整分析方向和重点。

```
### 比×υ：方向调整

**当前方向审视**：
现有重点：[拆分+探索+验证指向的主要方向]
隐含假设：[分析过程中没有明确但存在的假设]
偏向性：[分析是否有系统性偏向？]

**方向调整候选**：

**调整1 - 重新聚焦**：
原方向：[当前重点关注什么？]
调整为：[基于质疑，应该重点关注什么？]
理由：[为什么需要这个调整？]

**调整2 - 视角切换**：
原视角：[从什么角度分析问题？]
调整为：[是否需要换个角度？stakeholder/时间跨度/分析层次]
理由：[什么质疑驱动了视角切换？]

**调整3 - 边界重定**：
原边界：[问题分析的范围边界]
调整为：[扩大/缩小/重新划定边界]
理由：[边界问题如何影响结论？]

**调整4 - 优先级重排**：
原优先级：[探索/验证发现的重点排序]
调整为：[基于元质疑的新排序]
理由：[什么改变了重要性判断？]

**方向调整决策**：
□ 保持当前方向，质疑无效
□ 微调重点，保持框架
□ 重大调整，需要重构分析  
□ 完全转向，当前分析无效
```

**调整原则**：方向调整必须基于Step 1的具体质疑，不能是无根据的改变。

---

### Step 3: 排×υ (组织自指层)  
**功能表定位**：υ行×排列 = "质疑并调整现有结构"

调整分析结构，重新组织前序发现。

```
### 排×υ：结构重组

**当前结构审视**：
拆分结构：[ABCD四维度的组织逻辑]
探索结构：[高中低置信的可能性组织]
验证结构：[4×4矩阵的完备性逻辑]
整体架构：[三个阶段如何衔接？]

**结构问题识别**：
1. **维度问题**：[ABCD是否真正正交？某维度是否多余？]
2. **层次问题**：[信息是否在正确的抽象层次？]
3. **完整性问题**：[结构是否遗漏了关键要素？]
4. **一致性问题**：[不同阶段的逻辑是否一致？]

**重组方案**：

**方案1 - 维度重组**：
保留维度：[哪些维度经过质疑仍然有效？]
合并维度：[哪些维度其实是同一层次？]
新增维度：[质疑过程中发现的被忽略维度]
重新定义：[需要重新表述的维度]

**方案2 - 层次重组**：  
上移层次：[哪些分析应该在更高抽象层次？]
下移层次：[哪些需要更具体的层次？]
跨层整合：[不同层次的发现如何整合？]

**方案3 - 逻辑重组**：
因果重排：[重新梳理因果关系]
时序重排：[重新组织时间顺序]
重要性重排：[重新排列优先级]

**结构调整决策**：
□ 保持当前结构
□ 微调组织方式
□ 重大结构重组
□ 推翻重来

**重组输出**：
[如果选择调整，输出调整后的新结构]
```

---

### Step 4: 看×υ (观察自指层)
**功能表定位**：υ行×看列 = "汇总所有质疑，确认行动方案"

综合前三步质疑和调整，形成最终行动方案。

```
### 看×υ：行动方案确认

**质疑汇总**：
元质疑影响：[Step 1的质疑如何改变了问题理解？]
方向调整影响：[Step 2的调整如何影响了重点？]
结构重组影响：[Step 3的重组如何改变了分析框架？]

**置信度评估**：

**𝔇 (分析置信度)**：基于整个K4过程，对分析结果的信心
- 拆分质量：S₂ __/3, K3 __/4, 𝔑 __/4 → 拆分𝔇 ___
- 探索质量：信息充分度 + 发现意外性 → 探索置信度 ___  
- 验证质量：完备度 + 事实强度 → 验证置信度 ___
- 收敛质量：质疑深度 + 调整合理性 → 收敛置信度 ___

综合𝔇：___ (0-1)

**𝔎 (行动准备度)**：实际执行行动的准备程度  
- 明确性：目标/步骤是否清晰？ ___
- 可行性：资源/能力是否充分？ ___
- 时机性：当前是否合适时机？ ___  
- 风险性：潜在风险是否可控？ ___

综合𝔎：___ (0-1)

**行动方案**：

**核心结论**：
[经过完整K4分析和质疑调整后，对原始问题的核心判断]

**具体行动**：
1. **立即行动** (高确信，高准备度)：[具体可执行步骤]
2. **准备后行动** (高确信，中等准备度)：[需要什么准备？]
3. **进一步分析** (中等确信)：[需要补充什么分析？]
4. **暂缓决策** (低确信或低准备度)：[等待什么条件？]

**风险预案**：
- 主要风险：[最可能出现的问题]
- 监控指标：[如何知道是否偏离预期？]
- 调整机制：[出现问题时如何调整？]

**循环决策**：
□ 停止，执行行动 (𝔇≥0.8, 𝔎≥0.7)
□ 继续K4循环 (𝔇＜0.5, 轮数＜3)
□ 补充探索 (𝔎＜0.5, 信息不足)
□ 重新拆分 (根本方向有问题)
□ 改变框架 (K4不适合此问题)
```

```markdown
### Step 3: 排×υ 结构重组
[当前结构审视 + 问题识别 + 重组方案 + 调整决策]

### Step 4: 看×υ 行动方案确认
[质疑汇总 + 𝔇𝔎评估 + 行动方案 + 风险预案 + 循环决策]

---
**收敛总结**：
- 分析置信度𝔇：___，行动准备度𝔎：___
- 核心结论：[一句话总结]  
- 主要行动：[优先级最高的行动]

**kernel 探针**：如果现在搜一下，这个结论还成立吗？不是真搜——是问自己：收敛结论中有没有依赖"没有反面证据"而非"有正面证据"的部分？有 = 该部分在 kernel 里，置信度打折。

**多轮对比**（必填，首轮写"首轮，无上轮可比"）：
```
与上轮对比: [结论变了/没变] [行动方案变了/没变]
𝔇: X.XX (Δ𝔇 = +/-X.XX)
```

**路由指令**（标准格式，agent 直接执行）：
```
Cauchy 停止: 结论没变 AND 行动没变 AND 𝔇≥门槛 → 停止
    (门槛: 0.8 默认 / 0.9 BOOTSTRAP 2+信号)
继续:  𝔇<门槛 或 结论/行动有变 → 回退拆（新一轮）
病态:  Δ𝔇<0 连续两轮 → 强制停止（"问题需重新定义"）
       本轮≈前前轮≠前轮 → 强制停止（振荡，输出两方案）
       Δ𝔇≈0 连续两轮且𝔇<门槛 → 强制停止（"信息不足"）
保底:  轮数≥5 → 强制停止，输出当前最优
```
输出写入 `memory/k4_state.md`。
```

## 传递协议

**如果继续循环**：
- 输出重新定义的问题P'（基于质疑调整）
- 传递调整后的结构框架
- 说明循环的理由和目标

**如果停止**：
- 确认最终行动方案
- 说明停止的理由（置信度足够/轮数限制/框架不适用）

## 内生判断指南

### 纵向环特征

收敛是纵向环的终点，不搜索外部信息，纯基于内生判断：

```
✅ 内生认知模式：
- 基于已有分析进行质疑
- 调整基于逻辑一致性
- 置信度基于内在推理
- 行动基于综合判断

❌ 不做外部搜索：
- 不搜索新的支持/反对证据
- 不查询实施案例
- 不验证外部可行性
- 不获取新的事实信息
```

### 反模式预警

❌ **伪质疑**：表面质疑但实际确认原有结论  
❌ **过度质疑**：质疑一切导致分析瘫痪
❌ **跳过调整**：质疑发现问题但不调整结构/方向  
❌ **外部搜索**：收敛阶段搜索外部信息破坏内生模式
❌ **无限循环**：𝔇𝔎永远不满足，拒绝收敛
