# 验证 (validate)

> 碱基旋转：π→γ→υ→α (排→看→问→比)
> 功能表行：结构层（π行）
> 职责：外部吻合。格子和外部事实对得上吗？内部一致性是收的事，验不管。
> 前提（Gödel 完备性）：K4 意义完备 → 所有真命题可证 → 格子空 = 不完备。
> 极限（Gödel 不完备性）：完备 + 一致不可兼得。验发现矛盾时不是 bug——是系统极限。

**核心任务**：强制填满 4×4 矩阵。每格找外部事实，判断吻合度。填不上 = unknown unknowns。

**认知定位**：拆 = known knowns，探 = known unknowns，验 = unknown unknowns。验是涌现层。

**输入要求**：拆分+探索路径的全部输出

## 旋转执行：π→γ→υ→α

### Step 1: 排×π (组织结构层)
**功能表定位**：π行×排列 = "搭 4×4 必填矩阵"

```
### 排×π：4×4 矩阵

**问题维度**（来自拆）：A, B, C, D
**分析维度**（K4固定）：γ存在、π结构、α区分、υ自指

|  | A:[名] | B:[名] | C:[名] | D:[名] |
|---|---|---|---|---|
| **γ** | A×γ | B×γ | C×γ | D×γ |
| **π** | A×π | B×π | C×π | D×π |
| **α** | A×α | B×α | C×α | D×α |
| **υ** | A×υ | B×υ | C×υ | D×υ |

**逐格定义**：每格写一句话——这个交叉意味着什么。

16格全部必填。写不出来 = 你发现了一个 unknown unknown。
不是"这格没意义"——K4意义完备，每格都有意义。
写不出来是你还没看到，不是它不存在。
反本能去做。最难填的格子往往最有价值。

**填不上的缓冲机制**：
- 首次填不上 → 标记"待探索"，继续。回探时优先补这个格子
- 二次仍填不上（探过了还是空）→ 考虑K3退化：去掉这个维度，剩下三个是否仍然覆盖问题？
  是 → 维度可能多余，回拆缩减
  否 → 维度必要但信息真空，标记为致命盲点
```

---

### Step 2: 看×π (观察结构层)
**功能表定位**：π行×看列 = "逐格填事实，打吻合度"

为每格找对应的事实（从探的结果 + 新搜的确定性事实），判断吻合程度。

```
### 看×π：逐格填充与吻合度

**事实确定性**（用于判断能不能填进格子）：
- **确定**：多源交叉验证
- **条件确定**：单源 primary（原始研究/亲历），标记来源
- **待验证**：单源 secondary，不算确定，不能标 ●
- **不是事实**：观点、预测、理论推演，不能填格子

**逐格填充**：

| 格子 | 事实 | 吻合度 |
|---|---|---|
| A×γ | [找到的事实] | ●吻合 / ◐偏移 / ○填不上 |
| A×π | [找到的事实] | ●/◐/○ |
| A×α | ... | |
| A×υ | ... | |
| B×γ | ... | |
| ... | | |

**两层打分**：

**第一层：能不能填（unknown unknowns 检测）**
- ● 能填：找到了对应事实
- ○ 填不上：搜不到、想不到、没人做过 = unknown unknown

○ 不跳过。强制写出：
- 为什么填不上？
- 如果有答案会改变结论吗？
- 去哪找？

**第二层：发展到哪了（● 和 ○ 都打 0-1 分）**
- 1.0：事实和定义能互相推导。换一个事实还是吻合
- 0.7-0.9：事实支撑强，有小偏移但方向对
- 0.4-0.6：有相关事实但对不准。理解有偏差或维度定义需调
- 0.1-0.3：勉强能填，事实很弱或关联很远
- 0.0：完全填不上（○）

分数的意义不是"完备程度"——是**这个格子发展到哪了**。
16个分数组成的矩阵形状本身是诊断工具。
```

**4×4 吻合度矩阵**：

```
|  | A | B | C | D |
|---|---|---|---|---|
| **γ** | ● | ◐ | ○ | ● |
| **π** | ● | ● | ◐ | ◐ |
| **α** | ◐ | ● | ● | ○ |
| **υ** | ○ | ◐ | ○ | ● |
```

---

### Step 3: 问×π (质疑结构层)
**功能表定位**：π行×问列 = "质疑吻合度判断"

```
### 问×π：质疑

**1. ○ 格子深挖（unknown unknowns）**
这是验最有价值的产出。每个 ○ 格子：

| ○格子 | 为什么填不上 | 如果有答案会改变结论吗？ | 去哪找 |
|---|---|---|---|
| [格子] | [原因] | [是/否/不确定] | [方向] |

会改变结论的 ○ = 致命盲点。

**2. ◐ 格子诊断（偏移分析）**
每个 ◐：偏在哪？

| ◐格子 | 事实说的 | 格子定义说的 | 偏移方向 |
|---|---|---|---|
| [格子] | [事实] | [定义] | [是维度定义偏了还是事实不够精确？] |

维度定义偏了 → 回拆微调
事实不够精确 → 回探补搜

**3. ● 格子全检（Type I error — 假阳性是验最大的隐性风险）**
每个 ● 格子做反向推导：
- 从事实能不能推回格子定义？（能 = 真●，不能 = 假●，降为◐）
- 事实是 primary 还是 secondary？
- 换一条不同来源的事实还能吻合吗？

不是随机抽检。●全检。假阳性比○更危险——你以为覆盖了但没覆盖。

**4. 流浪事实**
探的结果里有没有放不进任何格子的事实？

流浪事实数：___
≥ 3 → 拆的维度有结构性遗漏。回拆。

**5. 外部出口**
验完后如果 ● 很多但总感觉不对 → 请求外部视角复检（永杰/其他模型）。
验检验验 = 自指循环，没有外部出口。元评估学科的解法：独立第三方。
不是每次都需要。但标记为可选逃生舱。
```

|  | A | B | C | D | 行均 |
|---|---|---|---|---|---|
| **γ** | _ | _ | _ | _ | _ |
| **π** | _ | _ | _ | _ | _ |
| **α** | _ | _ | _ | _ | _ |

验只产出这个矩阵。矩阵的形状分析、规律发现、瓶颈定位是收的工作。

**框架质量判定**（验的出口门控）：
- ○ = 0 → 进收
- ○ ≤ 2 且无致命盲点 → 进收（带盲点）

- 流浪事实：___条

## 标准化输出

1. 16格吻合度（●/○ 每格）
2. 发展度矩阵（0-1 每格 + 行均列均）
3. ○格子深挖（为什么填不上 + 会不会改变结论）
4. 流浪事实清单
5. ●全检结果（假阳性标记）

## 反模式预警

❌ **跳过○格子**：填不上就跳过。○是验最有价值的产出，不是障碍
❌ **删格子**：K4意义完备，16格都有意义。写不出来是你的盲点不是格子的问题
❌ **观点充数**：拿观点填○假装●。宁可留○也不造假
❌ **吻合度通胀**：大部分标●。真正的●能互相推导——格子定义推出事实，事实推出格子定义
❌ **框架审查事实**：带着"这格应该有什么"去搜。应该是"事实在这格吗？合不合？"
