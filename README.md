# K4 结构化思考框架

基于 Klein 四元群代数的问题分解体系。任意领域问题编译为 4 个正交维度，经 S₂ 配对检验和 K3 退化测试验证完备性。

## 这是什么

四个原子操作（看/排/比/问）× 四条思考路径（拆/探/验/收）= 16 个最小操作单元。

四条路径不是四个工具——是同一个分形结构的四种旋转，构成 Klein 四元群 V₄。任意两条路径组合等于第三条，每条路径执行两次回到原位。**这意味着四条路径不是任意选择，它们是唯一满足群封闭性的四个排列。**

```
拆 (1,2,3,4)  ←→  收 (4,3,2,1)    纵向环：不搜索，内生认知
探 (3,4,1,2)  ←→  验 (2,1,4,3)    横向环：外部搜索
```

## 架构

触发和路由不在调度器里——调度器是参考手册，路由由每个子 skill 自己控制。

```
BOOTSTRAP（数不可逆信号→决定深度）
    ↓
k4-think-decompose（拆：4维度 + S₂/K3 验证 + 𝔇）
    ↓ 路由指令：𝔇≥0.8 停，<0.8 继续
k4-think-breadth（探：外部搜索 + 可能性空间）
    ↓ 路由指令：充分停，有缺口继续
k4-think-completeness（验：4×4 矩阵 + 确定性事实）
    ↓ 路由指令：缺口<25% 继续，否则回退
k4-think-depth（收：元质疑 + 收敛行动）
    ↓ 路由指令：𝔇≥0.8 且 𝔎≥0.7 停
```

深度是自适应的——不预选模式，𝔇 驱动升级：

```
拆 → 𝔇≥0.8 停 → 事后看是"快速"
拆 → 探 → 停 → 事后看是"标准"
拆 → 探 → 验 → 收 → 停 → 事后看是"完整"
```

## 核心概念

**碱基功能表**（4×4 = 16 格）：

|  | 看 γ | 排 π | 比 α | 问 υ |
|---|---|---|---|---|
| **γ 存在** | 拆·1 | 拆·2 | 拆·3 | 拆·4 |
| **π 结构** | 验·2 | 验·1 | 验·4 | 验·3 |
| **α 区分** | 探·3 | 探·4 | 探·1 | 探·2 |
| **υ 自指** | 收·4 | 收·3 | 收·2 | 收·1 |

每条路径读自己起始维度的那一行，按旋转顺序遍历列。

**验证机制**：
- **S₂ 配对**：四元素的三种二二分组都有非平凡意义
- **K3 退化**：缺任一元素系统出现可识别的能力缺失
- **CE 反查**：「这件事怎么死？」——从后果反推遗漏维度

详见 [k4_meta.md](k4-think/references/k4_meta.md)。

## 用法

这些是 [OpenClaw](https://github.com/openclaw/openclaw) AgentSkill 格式。放入 OpenClaw workspace 的 `skills/` 目录即可自动加载。也可作为任何 LLM agent 的 prompt 模板使用——每个 SKILL.md 都是自包含的指令集。
