# K4-think v2

从问题本身长出四个正交维度，然后验证拆法对不对的思考框架。

别的框架给你固定的格子（SWOT 的四格、MECE 的分类）。K4 不预设格子——维度从问题的张力中生长，然后用数学方法验证它们正交且完备。

## 怎么用

遇到一个不能靠直觉解决的问题：

**1. 拆**：把问题拆成四个维度。验证维度正交（不重叠）且必要（少一个就不行）。

**2. 探**：按四个信息源搜索——学术怎么说、工程怎么做、做过的人怎么看、别的领域有没有类似问题。

**3. 验**：拿搜到的事实去撞你的框架。16 个格子（4维度 × 4层）逐个填。填不上的就是盲点。

**4. 收**：检查内部一致性。维度之间矛盾吗？换条路走能到同一个结论吗？把结论压成一句话，展得开吗？

大多数问题拆完就够了。不需要每次都走完四步。

## 30 秒例子

问题："我们的产品应不应该开源？"

**拆**成四个维度：
- A. 商业模式影响（开源后怎么赚钱）
- B. 技术生态（社区贡献 vs 维护成本）
- C. 竞争壁垒（开源是否削弱护城河）
- D. 团队文化（团队有没有开源协作经验）

检查：四个维度正交吗？少一个行不行？有没有遗漏的致命维度？

如果置信度够 → 停，按这四个面做决策。不够 → 探（搜索），看学术怎么说、同行怎么做、做过的人怎么看。

## 什么时候用

问自己：这件事搞砸了能挽回吗？

- 能挽回 → 不用 K4，直觉就行
- 不太能 → 拆一下，看看有没有漏掉的角度
- 完全不能 → 走完整流程

## 为什么是 4 不是 3 或 5

验证任何命题需要四个前提：有东西被讨论、推理不矛盾、命题可区分、推理有方向。少一个验证就做不完。第五个要么归约为前四个，要么和验证无关。

不是约定。是证明。详见 `meta.md`。

## 安装

```bash
git clone https://github.com/weiming0201/k4-think.git
```

**给 AI agent 用**：把文件复制到 agent 的 skills/ 或 prompts/ 目录。每个 .md 都是自包含的指令集。

```bash
# Claude Code 示例
cp k4-think/*.md your-project/.claude/commands/
# 或者 skills/ 目录
cp -r k4-think/ your-project/skills/k4/
```

适配 [Claude Code](https://claude.ai/code)、OpenClaw、或任何支持自定义 prompt 的 LLM 工具。

**给人用**：读 `chain.md` 了解流程，遇到具体问题按步骤走。`output.md` 帮你把结果写成别人能看懂的东西。

**快速上手**：
1. 有一个问题
2. 问自己：搞砸了能挽回吗？不能 → 用 K4
3. 读 `chain.md` → 读 `decompose.md` 开始拆
4. 拆完不确定 → 继续 `explore.md` → `validate.md` → `converge.md`
5. 每步做完看 `antichain.md` 对应的追问
6. 结果给别人看 → `output.md`
7. 想懂为什么 → `meta.md`

## 什么时候不该用

- 问题是线性的（步骤 1→2→3，没有分支）
- 有成熟框架直接能用（MECE 够了就用 MECE）
- 找不到第四个维度（可能问题不需要四维）
- 需要快速决策（K4 需要时间，不适合秒级判断）
- 元素之间有明确层级（A>B>C>D 不是正交的）

K4 不是万能的。过度使用 K4 = 分析瘫痪。

## 和其他框架的关系

| 框架 | 做什么 | 和 K4 的区别 |
|------|--------|-------------|
| **MECE** | 不重不漏地分类 | 告诉你"不重不漏"但不告诉你怎么拆、拆得对不对 |
| **SWOT** | 四格分析（优劣势/机会威胁） | 格子是固定的。K4 的维度从问题长出来 |
| **DSRP** (Cabrera) | 四种思维操作（区分/系统/关系/视角） | 最接近的。DSRP 描述思维的四种操作，K4 提供完整的操作流程和验证机制 |
| **CoT/ToT/GoT** | LLM 推理路径优化 | 管"怎么想"。K4 管"怎么看"。正交互补，可以组合 |
| **Integral/AQAL** (Wilber) | 四象限哲学框架（内外×个集） | 哲学层面有共鸣。K4 额外提供形式证明和操作流程 |

K4 的独特之处：维度从问题长出来（不是预设的格子），有形式证明（为什么是 4），有完整操作流程（拆探验收 + 调度 + 逆链），有收敛保证（Cauchy 判据）。
