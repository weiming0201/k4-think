# K4 调度器

> 理论基础：`skills/k4-think/references/k4_meta.md`
> 旋转修正：Full Row Passing + 环=信息获取模式

## 入口门控

```
P 是线性的（步骤 1→2→3）？ → 不用 K4
P 有成熟框架（MECE/SWOT/等）？ → 不用 K4
P 找不到 4 个独立元素？ → 不用 K4
P 一句话能答？ → 不用 K4
```

全否 → 进 K4，选模式。

## 四种模式

| 模式 | 路径组合 | 适用场景 | 信息流 |
|---|---|---|---|
| **快速** | 拆 | 只需四维度结构 | 纵向，不搜索 |
| **内省** | 拆→收 | 纯内生思考循环 | 纵向环，完整循环 |
| **标准** | 拆→探 | 需要外部信息 | 纵→横，单次切换 |
| **完整** | 拆→探→验→收 | 完备验证+收敛 | 全路径，双环切换 |

**默认：标准模式**

## 路由协议（重写）

### 传递机制：Full Row Passing

每条路径输出4步完整思考过程，下条路径接收**全部**4步输出作为上下文：

```
输入格式 → K4路径 → 输出格式
问题P → 拆(γ行) → {看已有, 排四维, 比正交, 问置信} + 四维度表
全输出 → 探(α行) → {比方向, 问prompt, 看结果, 排结构} + 可能性列表  
全输出 → 验(π行) → {排框架, 看打分, 问合理, 比缺口} + 完备矩阵
全输出 → 收(υ行) → {问元质, 比调向, 排调构, 看行动} + 收敛判断
```

**约束**：每条路径只能用前序路径的完整输出，不能访问原始问题P或外部上下文。

### 环切换：信息获取模式

**纵向环（拆↔收）**：内生认知，不搜索
- 快速：拆 → 停止
- 内省：拆 → 收 → 拆 → 收（最多3轮）

**横向环（探↔验）**：外部搜索
- 探：搜索可能性空间
- 验：搜索确定性事实

**跨环切换**：
- 拆→探：从纵向（内生）切到横向（搜索）
- 验→收：从横向（搜索）切回纵向（收敛）

### 路由决策

```
拆 完成，S₂≥2/3，𝔇≥0.7 →
  ├─ 快速模式 → 停止，输出四维度
  ├─ 内省模式 → 进收（纵向环）
  ├─ 标准模式 → 进探（跨环切换）
  └─ 完整模式 → 进探

探 完成，可能性≥3项 →
  ├─ 标准模式 → 停止，输出可能性列表
  └─ 完整模式 → 进验（横向环）

验 完成 → 反馈决策：
  ├─ 🔴Kill发现 → 强制中间收（跳过剩余路径）
  ├─ 缺口＜25% → 进收
  ├─ 理论框架有问题 → 回拆
  └─ 信息不足 → 回探

收 完成 → 𝔇×𝔎 决策：
  ├─ 𝔇≥0.8, 𝔎≥0.7 → 停止
  ├─ 𝔇＜0.5 → 回拆（新一轮）
  ├─ 𝔎＜0.5 → 回探（补信息）
  └─ 轮数≥3 → 强制停止
```

## 接口合约（重写）

### 碱基功能表映射

每条路径严格读取指定行，按旋转顺序访问列：

|  | 看γ | 排π | 比α | 问υ |
|---|---|---|---|---|
| **γ存在** | 拆1:回看已有 | 拆2:K4拆分 | 拆3:验证正交 | 拆4:判断置信 |
| **π结构** | 验2:搜索打分 | 验1:搭建框架 | 验4:找最大缺口 | 验3:验证合理 |
| **α区分** | 探3:汇总可能 | 探4:组织结构 | 探1:辨别方向 | 探2:写搜索prompt |
| **υ自指** | 收4:确认行动 | 收3:调整结构 | 收2:调整方向 | 收1:元质疑问题 |

### 旋转执行序列

```
拆：γ→π→α→υ (1,2,3,4) = 看→排→比→问
探：α→υ→γ→π (3,4,1,2) = 比→问→看→排  
验：π→γ→υ→α (2,1,4,3) = 排→看→问→比
收：υ→α→π→γ (4,3,2,1) = 问→比→排→看
```

每步读取：路径起始维度行 × 当前旋转位置列

### 输出标准化

每条路径必须输出：
1. **过程记录**：4步完整思考（按旋转序列）
2. **结构化输出**：标准格式的核心产出
3. **置信度**：路径执行的质量评估
4. **路由指令**：下一步建议（继续/回退/停止）

## 执行约束

1. **思行分离**：K4完成前不执行外部动作
2. **中间存档**：每步写入`memory/session_scratch.md`
3. **轮数限制**：最多3轮，第3轮强制收敛
4. **传递约束**：严格按full row passing，不跳步
5. **模式锁定**：选定模式后不可切换

```
你是K4思考执行器。

问题：{P的完整描述}
模式：{快速/内省/标准/完整}

严格按序执行：
1. 读 skills/k4-think-decompose/SKILL.md，执行拆分
2. [标准/完整] 读 skills/k4-think-breadth/SKILL.md，基于拆的全输出执行探索
3. [完整] 读 skills/k4-think-completeness/SKILL.md，基于拆+探全输出执行验证
4. [内省/完整] 读 skills/k4-think-depth/SKILL.md，基于前序全输出执行收敛

约束：每步完成后写中间结果到memory/，然后读取下一个skill
传递：向后续步骤传递前面所有步骤的完整输出，不只是最终结果
```
