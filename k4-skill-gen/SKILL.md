# K4 Skill 生成器

> 理论基础：`skills/k4-think/references/k4_meta.md`
> 四联模板：`references/quad_template.md`

## 入口：选档

```
用户给了领域描述 →
  需要几个skill？
  ├─ 1个 → 单文件档
  └─ 4个协调的 → 四联档（先完成单文件档再扩展）
```

四联档必须先完成单文件档。单文件档的四维度是四联档的基础。

---

## 单文件档

### Phase 1：领域K4拆分

对领域本身做拆分（不是对"怎么做skill"做拆分）。

**1a. 看：领域已有事实**
- 这个领域做什么？核心操作是什么？
- 已有工具/框架/约束？
- 典型输入→输出？

**1b. 排：拆四维度**

从领域的核心张力出发，拆出4个正交维度。

命名约束：
- 必须用领域术语，不能用万能词（"准确性/完整性"等）
- 每个维度对应领域内一类**独立决策**

**1c. 比：S₂ + K3 验证**

| 检验 | 通过标准 |
|---|---|
| S₂ 三配对 | 每组可一词命名 + 反转明显错误 |
| K3 退化 | 缺任一维度→skill 有明确能力缺失 |
| 𝔑 非显然 | 第4维度不是前3个的显然推论 |

S₂ < 2/3 → 重拆。K3 < 3/4 → 某维度多余，砍掉重来。

**1d. 问：选旋转**

默认 γ→π→α→υ（看→排→比→问）。选择依据：

| 旋转 | 适用场景 |
|---|---|
| γ→π→α→υ | 从观察开始，结构化输出。通用默认 |
| α→υ→γ→π | 从区分开始，搜索导向。信息收集类 |
| π→γ→υ→α | 从框架开始，验证导向。审计/检查类 |
| υ→α→π→γ | 从质疑开始，收敛导向。决策/评估类 |

### Phase 2：写 SKILL.md

```yaml
---
name: {skill-name}
description: "{领域}：{一句话功能}。四步流程：{维度1}→{维度2}→{维度3}→{维度4}。触发词：{3-5个}。"
---
```

**正文结构：**

```markdown
# {Skill名}

> 领域四维度：{A}, {B}, {C}, {D}
> 旋转序列：{碱基序列} ({中文序列})

## Step 1: {碱基}×{维度} — {动词短语}

**做什么**：{具体操作}
**输入**：{从哪来}
**输出格式**：{结构化模板}
**约束**：{硬边界}

## Step 2: {碱基}×{维度} — {动词短语}
[同上格式]

## Step 3: {碱基}×{维度} — {动词短语}
[同上格式]

## Step 4: {碱基}×{维度} — {动词短语}
[同上格式]

## 完整输出格式
[4步汇总模板]

## 反模式
[3-5条领域特有的反模式]
```

**Step 写法原则：**
- 每步 ≤ 150 词。够用就停
- 输出格式用代码块，可直接填
- 约束用硬标准（"不允许X"），不用软建议（"尽量避免X"）
- Step 之间有显式数据流：Step N 的输出 = Step N+1 的输入

### Phase 3：自检

| 检查 | 方法 |
|---|---|
| 4步覆盖领域核心操作？ | 对照 1a 的"核心操作"列表 |
| 步骤间无重叠？ | 每步输出独立，不重复前步 |
| 碱基×维度标签准确？ | 看=观察不分析，排=组织不创造，比=区分不搜索，问=质疑不执行 |
| 总长 < 500 行？ | 超了 → 拆到 references/ |

通过 → 输出 SKILL.md。不通过 → 回 Phase 2 修。

## 四联档

前提：单文件档已完成（有4个已验证维度）。

### Phase 4：生成四路径

从单文件的维度表生成 4×4 功能表：

```
|  | 看γ | 排π | 比α | 问υ |
|---|---|---|---|---|
| {维度A} (γ行) | [路径1·S1] | [路径1·S2] | [路径1·S3] | [路径1·S4] |
| {维度B} (π行) | [路径3·S2] | [路径3·S1] | [路径3·S4] | [路径3·S3] |
| {维度C} (α行) | [路径2·S3] | [路径2·S4] | [路径2·S1] | [路径2·S2] |
| {维度D} (υ行) | [路径4·S4] | [路径4·S3] | [路径4·S2] | [路径4·S1] |
```

维度→行分配：哪个维度最"存在性"→γ行，最"结构性"→π行，最"区分性"→α行，最"自指性"→υ行。

**16格内容生成规则：**
- 格 = 碱基操作 × 维度。两者交叉产生一个具体操作
- 每格一句话描述，不超过 20 字
- 16格必须互不相同

### Phase 5：写4个 SKILL.md + 调度器

**四路径映射（固定）：**

| 路径 | 文件名后缀 | 读哪行 | 旋转 | 置换签名 |
|---|---|---|---|---|
| 路径1 | `-init` | γ行(A) | γ→π→α→υ | (1,2,3,4) |
| 路径2 | `-search` | α行(C) | α→υ→γ→π | (3,4,1,2) |
| 路径3 | `-verify` | π行(B) | π→γ→υ→α | (2,1,4,3) |
| 路径4 | `-converge` | υ行(D) | υ→α→π→γ | (4,3,2,1) |

命名：`{skill-name}-init`, `{skill-name}-search`, `{skill-name}-verify`, `{skill-name}-converge`

**后缀可换**。`-init/-search/-verify/-converge` 是默认。如果领域有更自然的术语，用领域术语。但必须在调度器中注明映射关系。

**每个 SKILL.md 的结构：** 同单文件档 Phase 2 格式，额外加：

```markdown
> 功能表行：{维度}层（{γ/π/α/υ}行）
> 信息模式：{纵向环/横向环}，{内生/外部搜索}

**输入要求**：{前序路径的全部输出 / 问题P}
**输出传递**：全部4步思考过程 → {下游路径}的完整上下文
```

**环分配（固定）：**

| 环 | 路径 | 信息模式 |
|---|---|---|
| 纵向 | init + converge | 不搜索外部 |
| 横向 | search + verify | 搜索外部信息 |

search 搜可能性，verify 搜确定性。

**调度器 SKILL.md：** 见 `references/quad_template.md`

### Phase 6：一致性审计

10项硬检查，全过才输出：

| # | 检查 | 方法 |
|---|---|---|
| 1 | 旋转序列 | 4文件 × 3处（头部/description/正文）一致 |
| 2 | 功能表映射 | 16格 Step 号 vs 功能表标记一致 |
| 3 | 置换签名 | 4个签名构成 Klein 群 V₄ |
| 4 | 镜像对 | init↔converge 互逆，search↔verify 互逆 |
| 5 | 自然续接 | 前路径终止维度 = 后路径起始维度 |
| 6 | 环分配 | 纵向不搜索，横向搜索 |
| 7 | Full Row Passing | 输入逐级累积，不跳步 |
| 8 | 碱基×维度标签 | 行固定，列按旋转遍历 |
| 9 | 调度器功能表 | 与 k4_meta 功能表结构一致 |
| 10 | description 一致 | frontmatter 与正文匹配 |

失败项 → 定位修复 → 重跑审计。

## 反模式

❌ **跳过单文件直接写四联**：维度没验证就展开，4倍工作量返工
❌ **万能维度**：四个维度对换到别的领域也成立 → 没有领域特异性
❌ **碱基当维度**：把看排比问当成skill的四维度 → 混淆操作和对象
❌ **16格硬凑**：某些交叉格没有自然含义 → 维度行分配有问题，回 Phase 4
❌ **超长Step**：单步 > 150 词 → 拆到 references 或简化
