# 验证 (validate)

> 碱基旋转：π→γ→υ→α (排→看→问→比)  
> 功能表行：结构层（π行）  
> 信息模式：横向环，确定性搜索  
> 理论基础：`skills/k4-think/references/k4_meta.md`

**核心任务**：搭建4×4完备性框架，填充确定性事实，识别最大缺口。

**输入要求**：拆分+探索路径的全部输出（完整上下文传递）

**输出传递**：全部4步思考过程 → 收敛路径的完整上下文

## 旋转执行：π→γ→υ→α

### Step 1: 排×π (组织结构层)
**功能表定位**：π行×排列 = "基于K4理论，拆分结构维度"

基于K4理论建立4×4矩阵框架，与拆分的问题维度交叉。

**输入解析**：
```
从拆分输出提取问题维度：[A, B, C, D]
从探索输出提取可能性空间：[高置信X个，中等Y个，低置信Z个]
```

**框架构建**：
```
### 排×π：4×4完备性框架

**问题维度**（来自拆分）：A, B, C, D
**分析维度**（K4结构）：γ存在、π结构、α区分、υ自指

|  | A:[维度名] | B:[维度名] | C:[维度名] | D:[维度名] |
|---|---|---|---|---|
| **γ存在** | A×γ | B×γ | C×γ | D×γ |
| **π结构** | A×π | B×π | C×π | D×π |
| **α区分** | A×α | B×α | C×α | D×α |
| **υ自指** | A×υ | B×υ | C×υ | D×υ |

**格子定义**：
- A×γ: [A维度在存在层面的具体含义]
- A×π: [A维度在结构层面的具体含义]  
- A×α: [A维度在区分层面的具体含义]
- A×υ: [A维度在自指层面的具体含义]
[继续定义所有16格]

**框架检验**：每格都有独立含义且不可从其他格推导？
```

**理论约束**：γπαυ是固定的分析维度，不随问题改变。问题维度ABCD与分析维度的交叉必须产生16个独立有意义的格子。

---

### Step 2: 看×π (观察结构层)  
**功能表定位**：π行×看列 = "搜索确定性事实，逐格打分"

为每个格子搜索确定性事实并评分。

```
### 看×π：确定性事实搜索

**A×γ事实搜索**：
查询："[具体查询词，针对A维度的存在层面]"
确定性发现：[可验证的事实，不是观点]
证据强度：[强/中/弱/无]
置信度：[0-1分]

**A×π事实搜索**：
查询："[A维度的结构层面查询]"
确定性发现：[框架性、规律性发现]  
证据强度：[强/中/弱/无]
置信度：[0-1分]

[继续为所有16格搜索...]

**4×4打分矩阵**：

|  | A | B | C | D |
|---|---|---|---|---|
| **γ** | 0.8 | 0.6 | 0.3 | 0.1 |
| **π** | 0.7 | 0.9 | 0.4 | 0.2 |  
| **α** | 0.5 | 0.7 | 0.8 | 0.6 |
| **υ** | 0.2 | 0.4 | 0.5 | 0.3 |
```

**打分标准**：
- 1.0：充分确定性事实，多源验证
- 0.7-0.9：较强事实支持，少量不确定性
- 0.4-0.6：一定事实基础，需要更多验证  
- 0.1-0.3：事实薄弱，主要是推测
- 0.0：完全没有事实支持

**搜索约束**：只寻找可验证的确定性事实，不接受观点、推测、理论假设。

---

### Step 3: 问×π (质疑结构层)
**功能表定位**：π行×问列 = "验证维度和打分的事实合理性"

质疑框架设计和事实搜索的合理性。

```
### 问×π：合理性质疑

**框架质疑**：
1. 16格是否真的独立？[检查重叠的格子]
2. 某些格子是否无意义？[A×υ这种交叉是否强行组合？] 
3. 分析维度γπαυ是否适用？[某些问题域是否不适合4层分析？]

**打分质疑**：
1. 低分格子的搜索是否充分？[0.1-0.3的格子是真的没信息还是没找到？]
2. 高分格子是否过于自信？[0.8+的格子是否忽略了不确定性？]
3. 某维度整体偏低是否意味着维度定义有问题？

**事实质疑**：
1. "确定性事实"中有多少其实是包装过的观点？
2. 搜索查询是否有偏向性？[是否只搜索支持性证据？]
3. 多源验证是否真的独立？[不同来源是否引用同一根源？]

**框架外检测**：
搜索过程中是否出现不属于任何维度的致命事实？
- [ ] 列出所有"找不到归属格子"的事实
- [ ] 其中有致命级别的吗？→ 有 = 拆分遗漏维度，反馈回拆

**质疑评估**：
- 致命质疑：[会导致整个框架失效的问题]
- 重要质疑：[影响结论可靠性的问题]  
- 一般质疑：[需要注意但不影响整体的问题]
```

**合理性门槛**：如果致命质疑 >1个，建议回到Step 1重构框架或回到拆分重新定维度。

---

### Step 4: 比×π (区分结构层)
**功能表定位**：π行×比较列 = "对比矩阵空缺，找最大缺失方向"

分析完备性矩阵，识别知识缺口和优先补全方向。

```
### 比×π：缺口分析

**整体完备度**：
平均分：[16格平均] 
方差：[分布是否均匀]
充分阈值(≥0.7)：___/16格
薄弱阈值(≤0.3)：___/16格

**维度缺口排序**：
| 问题维度 | 平均分 | 最弱格子 | 缺口程度 |
|---|---|---|---|
| A | 0.55 | A×υ(0.2) | 中等 |
| B | 0.65 | B×γ(0.6) | 较小 | 
| C | 0.50 | C×γ(0.3) | 较大 |
| D | 0.30 | D×γ(0.1) | 严重 |

**分析维度缺口排序**：
| 分析维度 | 平均分 | 整体特征 |
|---|---|---|
| γ存在 | 0.45 | 事实搜索困难 |
| π结构 | 0.55 | 框架性知识尚可 |
| α区分 | 0.65 | 对比分析较强 |  
| υ自指 | 0.35 | 元层面信息稀缺 |

**最大缺口**：
1. **D×γ(0.1)**：D维度的存在层事实严重缺失
2. **A×υ(0.2)**：A维度的自指层信息极少
3. **C×γ(0.3)**：C维度的存在层事实薄弱

**补全建议**：
- 优先级1：[针对最大缺口的具体行动]
- 优先级2：[次要缺口的补全策略]
- 优先级3：[提升整体平均分的方向]

**反馈决策**：
□ 缺口<25%，信息充分 → 进收敛
□ 特定维度整体薄弱 → 回拆分（重新定义维度）
□ 事实搜索系统性困难 → 回探索（改变搜索策略）
□ 框架设计有根本问题 → 重新设计框架
```

## 搜索执行指南

### 确定性搜索原则

```
高质量确定性事实：
✅ 政府统计数据、学术研究结论、企业财报数字
✅ 历史事件、法律条文、技术规格  
✅ 可重复验证的实验结果

低质量或非确定性：
❌ 专家观点（除非广泛共识）
❌ 媒体解读、分析师预测
❌ 单一来源的调查结果
❌ 理论推演、模型预测
```

### 反模式预警

❌ **格子重叠**：A×γ和A×π说的是同一件事  
❌ **观点当事实**：把分析、观点、预测当作确定性事实  
❌ **搜索偏向**：只搜索支持性证据，忽略反面事实  
❌ **过度自信**：基于薄弱证据给高分
❌ **回避缺口**：不承认某些格子真的没有足够事实
