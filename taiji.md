# K4 太极（全局调度）

无极生太极，太极生两仪，两仪生四象，四象生八卦。

- **无极**：`meta.md` — 意义完备。K4 是什么、为什么
- **太极**：本文件 — 结构完备。能做什么、怎么选
- **两仪**：`chain.md`（阳）+ `antichain.md`（阴）— 正反循环调度
- **四象**：`decompose/explore/validate/converge` — 纯函数。input→output
- **八卦**：领域实例化 — TODO

## 入口

每层都是入口。选哪层取决于你有多少时间和需要多少确信。

```
问题P到来
  ↓
数不可逆信号（行动不可逆 / 误读代价致命 / 直觉无用 / 后果影响他人）
  0 → 不用K4
  1+ → 选入口层
```

不知道要什么 → 太极给你选：

| 选择 | 什么时候用 | 怎么走 |
|------|-----------|--------|
| **全遍历** | 需要最高完备度 | 正转+反转，8维度全走 |
| **正环** → chain.md | 有自己的判断 | 拆→探→验→收 |
| **逆环** → antichain.md | 没有自己的看法 | 探→拆→收→验 |
| **单维度** → 某个 skill | 有大量前置信息，从某个维度快速收束 | 进一个 skill → 对角线收束 |
| **成品 skill** → output 等 | 具体实践问题 | 直接调用八卦层 |

**空输入/模糊问题**：如果 P 不明确 → 先用拆的 Step 1（看×γ：观察已有事实）理清 P 是什么。拆本身能处理模糊输入。

```
        扩张    收缩
内部     拆      收
外部     探      验
```

四象速查的对角线：拆↔验（内部扩张撞外部收缩），探↔收（外部扩张撞内部收缩）。

## 𝔇 门槛

```
𝔇门槛 = 0.8 + 0.05 × 不可逆信号数（上限0.95）— Neyman-Pearson
  信号1 → 0.85
  信号2 → 0.90
  信号3+ → 0.95
```

## 新一轮标准

大流程不回退。一轮走完 𝔇 不够 → 新一轮（带本轮结果重开）。
"为什么不是另一个方案"答不出 → skill 内部处理（黑盒），不影响大流程。

## 核心规则

1. **大流程不回退。** 6 种路径（2 环 + 4 对角线）严格按序。skill 内部可以回退（黑盒）。𝔇 不够 → 新一轮
2. **𝔇 是轮末出口门。** 一轮走完看 𝔇 决定停还是新一轮
3. **停是默认。** 大多数问题一步或一轮就够
4. **单人完整回路**
5. **每步强制写下来。** 下一步在写入的结果上开始。不写 = 没做
6. **外部模型视角是工具不是协作**

## 四象标准化输出

每个 skill 是纯函数。任意输入 → 标准化输出。

| skill | 标准化输出 |
|---|---|
| **拆** | 四维度 + S₂/2×2/K3/𝔑/CE + 𝔇 |
| **探** | 四源结果 + 三件套 + 信息层级 + 主要意外 |
| **验** | 16格吻合度 + 发展度矩阵 + ○深挖 + 流浪事实 + ●全检 |
| **收** | 矩阵形状 + 矛盾 + PI + 压缩结论 + kernel探针 + 𝔇 |

## 执行机制

太极创建文件夹，一切在文件夹内操作。

### 目录结构

**正环或逆环**：
```
k4_runs/[日期]_[问题名]/
├── state.md           进度+摘要（从 state_template.md 复制）
├── decompose.md       拆的详细输出
├── explore.md         探的详细输出
├── validate.md        验的详细输出
└── converge.md        收的详细输出
```

**全量**（正+逆独立跑，最后比对）：
```
k4_runs/[日期]_[问题名]/
├── yang/
│   ├── state.md
│   ├── decompose.md → explore.md → validate.md → converge.md
│   └── ...
├── yin/
│   ├── state.md
│   ├── explore.md → decompose.md → converge.md → validate.md
│   └── ...
└── comparison.md      两环结论差异（最后生成）
```

**comparison.md 比对内容**：
1. 一句话结论是否相同
2. 维度是否相同（正环拆出的 vs 逆环拆出的）
3. 𝔇 差异
4. 正环发现但逆环没发现的（和反过来）
5. 结论相同 → 高确信。结论不同 → 两方案并列，标注差异来源

**速查**（四象入口，无 state）：
```
直接调用 skill → 生成 output → 触发对角线 skill → 完成
```

### 流程

```
1. 太极：选模式 → 创建目录 → 初始化 state.md
2. 阳/阴：读 state → 调用当前步骤的 skill
3. skill：读 state 里的摘要作为 context → 执行 → 写自己的 output 文件
4. skill 完成后：更新 state（进度+摘要+𝔇）
5. 阳/阴：读更新后的 state → 按步间调度决定下一步 → 回到 2
6. 结束：state 记录完整轨迹
```

skill 不管从哪来回哪去。skill 只读 state、写 output、更新 state。从哪来回哪去是阳/阴的事。

四象直接入口：没有 state 文件 → 跑完当前 skill → 触发对角线 → 速查完成。

## 反模式

❌ **为了跑K4而跑K4**：信号=0时强行启动
❌ **只知道正转**：没有看法时应该反转
❌ **skill 里写调度**：调度全在太极+两仪。skill 是纯函数
❌ **不建目录就跑**：输出散落各处无法追溯
